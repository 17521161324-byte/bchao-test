---

## 一、项目定位

本平台是辅助生殖 B 超语音识别的内部测试与评估工具。

核心目标不是直接替代生产系统，而是用真实检查录音、真实 B 超结果和不同模型组合，评估：

- 哪个 ASR 模型更适合当前业务语音；
- 哪个 LLM / 提示词组合能更准确抽取结构化字段；
- 不同模型组合在准确率、耗时、稳定性上的差异；
- 单条检查记录与批量实验结果是否可追溯、可复现。

## 二、核心业务口径

### 2.1 最小业务维度：检查记录

当前系统的最小业务单位是“检查记录”，而不是“患者”。

代码中 `patient_records.id` 实际表示一次检查记录 ID，可理解为：

```text
exam_record_id = patient_records.id
```

业务层级为：

```text
病历号 record_id
  └── 检查记录 exam_record / patient_record
        ├── 日期 date
        ├── 时间戳文件夹 timestamp_folder
        ├── 录音分段 audio_segs
        ├── B 超真实结果 b_ultra_results
        ├── ASR 不同模型转写结果 patient_asr_results
        └── LLM 结构化提取结果 patient_llm_results
```

说明：

- `record_id` 是病历号，可跨日期重复，不能作为唯一关联键。
- `patient_records.id` 才是检查记录主键。
- ASR、LLM、实验任务、评估结果都应以检查记录 ID 关联。

## 三、现有数据资产

### 3.1 录音文件

录音按日期、病历号、时间戳文件夹、分段文件组织：

```text
B超录音文件/
├── 20260622/
│   ├── A016675/<timestamp>/audio/seg-0001.wav
│   └── ...
├── 20260623/
├── 20260624/
└── 20260626/
```

特点：

- 原始录音为分段文件，不是单个完整音频；
- 每个检查记录可关联多个 `audio_segs`；
- 当前 ASR 执行以分段为单位，最终拼接为 `full_transcript`。

### 3.2 B 超真实结果

B 超结果来自 xlsx 导入，作为金标准。

核心字段：

| 字段 | 说明 |
|---|---|
| 病历号 | 展示与初步匹配字段，不作为唯一主键 |
| 日期 | 用于定位检查记录 |
| 右侧卵泡 | 明细与总数 |
| 左侧卵泡 | 明细与总数 |
| 内膜厚度 | mm |
| 内膜类型 | A/B/C/A-B 等 |
| 左右卵巢长宽 | mm |
| 备注 | 可选 |

## 四、当前功能规划

### 4.1 数据管理

目标：围绕“检查记录”管理录音、B 超真实值、ASR 转写结果和 LLM 提取结果。

已具备 / 正在迭代：

- 按日期筛选检查记录；
- 展示检查记录列表；
- 打开检查详情抽屉；
- 播放当前检查记录的音频分段；
- 展示 B 超真实值；
- 按 ASR 模型展示当前检查记录的转写状态；
- 点击不同 ASR 模型，展示对应转写内容；
- 已完成 ASR 结果可复用，避免反复调用；
- LLM 提取基于当前选中的 ASR 结果；
- 提示词模版管理弹窗，支持左侧切换、右侧查看编辑、保存和应用到当前检查。

### 4.2 模型配置

目标：统一管理 ASR / LLM 模型。

能力：

- ASR / LLM 模型增删改查；
- 配置 provider、endpoint、api_key、model_name、params；
- 支持 active / inactive；
- 连通性测试；
- 支持本地 FunASR、豆包/火山 ASR、MiMo ASR、DeepSeek/OpenAI 兼容 LLM 等接入形态。

### 4.3 提示词模版管理

目标：把 LLM 提示词从代码固定内容升级为可管理配置。

能力：

- 提示词模版列表；
- 新增、编辑、删除；
- 设为默认；
- 强制包含 `{transcript}` 占位符；
- 在检查详情中应用到当前 LLM 提取；
- 后续可扩展为提示词版本对比。

### 4.4 批量实验

目标：批量验证不同 ASR / LLM / 提示词 / 热词组合。

能力：

- 创建实验批次；
- 选择日期和检查记录范围；
- 配置实验组合；
- Worker 后台执行任务；
- ASR → LLM → 自动评估；
- 实验任务保存快照，保证历史结果可复现；
- 实验详情展示检查记录、ASR 转写、LLM 结构化结果、B 超真实值与评估结果。

## 五、数据结果策略

### 5.1 检查记录级结果

`patient_asr_results` 和 `patient_llm_results` 是检查记录级正式结果。

用途：

- 数据管理详情页长期展示；
- 页面刷新后不丢失；
- 同一检查记录可保存多个 ASR 模型结果；
- LLM 结果明确绑定某一次 ASR 结果；
- 后续实验可选择复用已有结果。

### 5.2 实验快照

`experiment_tasks` 保存某次实验使用的结果快照。

即使关联了：

- `asr_result_id`
- `llm_result_id`

仍然保留：

- `asr_results`
- `full_transcript`
- `structured_result`
- `evaluation`

原因：历史实验必须可复现，不能因为检查记录当前 ASR/LLM 被切换而改变。
