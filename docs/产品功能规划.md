## 一、项目背景

### 1.1 业务场景

辅助生殖科室中，患者在做排卵监测时需要通过 B 超检查卵泡发育情况。医生在现场一边操作 B 超一边口述观察结果（如卵泡数量、尺寸、位置等），这些口述信息需要被记录并导入业务系统。

### 1.2 目标

构建一套**语音识别处理链路**：现场录音 → ASR 文字转写 → LLM 语义纠错与结构化抽取 → 输出可用格式（字段 / 总结文本）→ 导入业务系统存储。

### 1.3 已完成工作

- ✅ 真实业务场景下患者录音的采集（排卵监测现场）
- ✅ 患者基本信息采集（唯一病历号作为主键）
- ✅ 患者当日 B 超检查的真实数据记录结果（作为评估金标准 / ground truth）

### 1.4 当前任务

搭建一个**内部测试平台**，用于对比以下变量对最终识别准确率的影响：

| 变量维度 | 对比内容 |
|---------|---------|
| 收音设备 | 手机、领夹麦、桌面麦克风等不同设备的收音效果差异 |
| ASR 模型 | 在线（讯飞、腾讯等） vs 本地（已部署 FunASR，端口 50000） |
| LLM 模型 | 在线（讯飞、腾讯等）；本地 LLM 暂未部署，后续扩展 |
| 提示词与热词 | 不同提示词版本、不同专业热词表的效果差异 |

最终目的：**通过量化对比，找到最优组合方案**（准确率 × 延迟 × 成本）。

## 二、现有数据资产

### 2.1 录音文件结构

```
B超录音文件/
├── 20260622/
│   ├── A016675/1782xxxxxxx/audio/seg-0001.wav ~ seg-0013.wav
│   ├── A017503/...
├── 20260623/
├── 20260624/
└── 20260626/
```

- 已按 **日期 → 病历号 → 分段 seg** 组织
- 格式：WAV，已切片为若干 seg（每个 seg 为一次连续录音片段）

### 2.2 B 超结果文件（ground truth）

每个日期对应一个 xlsx 文件，字段如下：

| 字段 | 说明 | 示例 |
|------|------|------|
| 病历号 | 唯一标识，与录音关联 | A017750 |
| 右侧卵泡数 | 格式：`尺寸×数量` 拼接 | `16.4×1  15.8×1...` |
| 左侧卵泡数 | 同上 | `16.3×1  15.2×1...` |
| 内膜厚度 | 数值（mm） | 6.4 |
| 内膜类型 | A / B / C / A-B 等 | A |
| 右卵巢长/宽 | 数值（mm） | 28 / 33 |
| 左卵巢长/宽 | 数值（mm） | 46 / 27 |
| 备注 | 可选文本 | 移植2枚囊胚... |

> 注：`×1` 表示该尺寸卵泡 1 个，`×2` 表示 2 个，以此类推。

### 2.3 本地 ASR 服务

- 地址：`http://172.16.10.142:50000`
- 接口：`POST /transcribe`，multipart/form-data，参数 `file`（WAV 16kHz 单声道）
- 返回：`{"transcript": "转写文字"}`
- 健康检查：`GET /health`

## 四、第一阶段功能详细设计（MVP）

### 4.1 页面 1：数据导入与管理

- 按 **日期 → 病历号 → seg** 三级树形结构展示录音
- 支持点击 seg 播放
- 上传 xlsx 结果文件，自动解析卵泡数、内膜等字段
- 展示数据匹配状态（已匹配 / 仅录音 / 仅结果）

### 4.2 页面 2：模型配置

- ASR / LLM 模型增删改查
- 配置名称、类型、Endpoint、API Key、Model Name
- 连通性测试（一键测试每个模型是否可用）
- 内置默认本地 FunASR 配置

### 4.3 页面 3：单条测试（核心）

- 选择病历号 → 选 ASR 模型 → 选 LLM 模型（可选）→ 开始测试
- SSE 实时推送 ASR 逐 seg 转写进度
- 音频播放器支持 seg 切换、自动连播
- LLM 后处理输出结构化字段 + 总结文本
- 自动评估准确率（如有 ground truth）

### 4.4 页面 4：结果评估

- 识别结果与真实值逐字段对比表格
- 差异项标红
- 人工修正功能（编辑结构化结果后重新评估）
- 字段级准确率统计

### 4.5 页面 5：测试历史

- 历史测试记录列表
- 按病历号筛选
- 查看详情（转写全文、总结、结构化结果）

## 六、技术方案

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | React 18 + TypeScript + Ant Design 5 + Vite | 生态成熟、组件丰富 |
| 后端 | Python 3.11+ + FastAPI + SQLAlchemy | 异步支持好、数据处理生态成熟 |
| 数据库 | SQLite（MVP）→ PostgreSQL | 轻量起步、SQLAlchemy 切换成本低 |
| 反向代理 | Nginx (Windows) | 统一入口、静态资源、SSE 支持 |
| 进程管理 | PM2 | 已具备、支持 Python 进程保活 |
| 部署 | 无 Docker，直接运行 | 适合 Windows 个人电脑 |

## 八、部署步骤（Windows）

### 前置要求
- Python 3.9+
- Node.js 18+
- PM2 (`npm install -g pm2`)
- Nginx

### 1. 后端部署
```bash
cd backend
pip install -r requirements.txt
copy .env.example .env
# 修改 .env 配置
pm2 start "uvicorn app.main:app --host 0.0.0.0 --port 8000" --name bchao-backend
pm2 save
pm2 startup
```

### 2. 前端部署
```bash
cd frontend
npm install
npm run build
# dist/ 产物
```

### 3. Nginx 部署
- 复制 `nginx.conf` 到 Nginx 配置目录
- 修改 `root` 和 `alias` 路径为实际路径
- 启动 Nginx

### 4. 访问
- 浏览器打开 `http://服务器IP`
