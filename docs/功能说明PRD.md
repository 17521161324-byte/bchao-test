# B 超语音测试平台 — 功能说明 PRD

> 创建日期：2026-07-07  
> 文档类型：现有功能说明 / PRD  
> 适用范围：数据管理、模型配置、提示词模版、检查详情、批量实验、结果评估  
> 当前口径：系统最小业务维度为“检查记录”

---

## 一、产品概述

本系统用于辅助生殖 B 超语音识别链路的内部测试、评估和模型选型。

系统围绕真实检查记录，管理：

- 原始录音分段；
- B 超真实检查结果；
- 不同 ASR 模型的转写结果；
- 不同 LLM / 提示词组合的结构化提取结果；
- 批量实验任务与结果快照；
- 字段级评估结果。

系统当前不是最终生产业务系统，而是用于验证“ASR + LLM + 提示词 + 热词”组合效果的内部测试平台。

---

## 二、核心业务定义

### 2.1 检查记录是最小业务维度

系统最小业务单位是“检查记录”，不是患者，也不是病历号。

代码中：

```text
patient_records.id = exam_record_id = 检查记录 ID
```

业务层级：

```text
病历号 record_id
  └── 检查记录 exam_record
        ├── 日期
        ├── 时间戳文件夹
        ├── 录音分段
        ├── B 超真实结果
        ├── ASR 转写结果
        └── LLM 结构化结果
```

说明：

- 同一个 `record_id` 可以在多个日期出现；
- `record_id` 只能用于展示、搜索和辅助匹配；
- ASR/LLM/实验任务必须基于 `exam_record_id` 关联；
- 前端当前 `selectedRecord.id` 就是检查记录 ID。

### 2.2 结果类型

| 结果类型 | 数据表 | 业务含义 |
|---|---|---|
| 录音分段 | `audio_segs` | 当前检查记录的原始音频 |
| B 超真实结果 | `b_ultra_results` | 人工整理或导入的金标准 |
| ASR 结果 | `patient_asr_results` | 某个 ASR 模型对当前检查记录的转写 |
| LLM 结果 | `patient_llm_results` | 某个 LLM 基于某次 ASR 的结构化提取 |
| 实验快照 | `experiment_tasks` | 某次批量实验运行时固定保存的结果 |

## 四、数据管理功能说明

### 4.1 检查记录列表

页面：`/data`

主要能力：

1. 按日期批次筛选检查记录；
2. 按病历号搜索；
3. 展示每条检查记录的：
   - 病历号；
   - 日期；
   - 是否有录音；
   - 是否有 B 超结果；
4. 点击“详情”或表格行进入检查详情抽屉。

### 4.2 检查详情

检查详情以当前检查记录为中心。

顶部展示：

- 病历号；
- 日期；
- 录音分段数量；
- 是否存在 B 超结果。

详情主体包含：

1. ASR 语音转写；
2. LLM 结构化提取；
3. 卵泡明细对比；
4. B 超真实结果；
5. 提示词模版管理。

## 五、ASR 语音转写功能说明

### 5.1 功能目标

在同一条检查记录中，允许多个 ASR 模型分别转写并保存结果。

用户可以：

- 查看所有启用的 ASR 模型；
- 判断每个模型是否已对当前检查记录完成转写；
- 点击不同模型查看不同转写结果；
- 对未转写模型发起转写；
- 对已转写模型选择重新识别。

### 5.2 ASR 模型状态

ASR 模型状态来自当前检查记录的 `patient_asr_results`。

| 状态 | 说明 | 页面含义 |
|---|---|---|
| 无记录 | 当前检查记录未使用该模型转写 | 未转写 |
| running | 转写任务创建但未完成 | 转写中 |
| success | 转写成功并已保存 | 已转写 |
| failed | 转写失败 | 失败 |

状态判断维度：

```text
exam_record_id + asr_model_id
```

不能使用：

```text
record_id + asr_model_id
```

### 5.3 ASR 展示逻辑

打开检查详情时：

1. 加载所有 active ASR 模型；
2. 调用当前检查记录 ASR 结果列表；
3. 按 `asr_model_id` 分组；
4. 每个模型优先展示最新 success 结果；
5. 没有 success 时展示最新 running/failed 记录；
6. 默认选中当前采用结果，若无则选中第一个有成功结果的模型。

### 5.4 ASR 调用逻辑

用户点击“开始识别/重新识别”：

```text
GET /api/patients/{exam_record_id}/asr/stream?asr_model_id={model_id}
```

后端流程：

1. 创建 `patient_asr_results`，状态为 `running`；
2. 读取当前检查记录的所有音频分段；
3. 按分段调用 ASR；
4. SSE 推送进度；
5. 拼接 `full_transcript`；
6. 保存分段结果和全文；
7. 状态更新为 `success`；
8. 设置为当前采用 ASR。

失败时：

- 状态更新为 `failed`；
- 保存 `error_message`；
- 页面展示失败状态和错误提示。

### 5.5 复用规则

如果当前检查记录已经存在某 ASR 模型的 success 结果：

- 页面默认展示已保存结果；
- 不自动重复调用模型；
- 只有用户主动点击“重新识别”才再次调用。

## 六、LLM 结构化提取功能说明

### 6.1 功能目标

LLM 提取必须基于当前选中的 ASR 结果。

用户操作路径：

1. 打开检查详情；
2. 选择或查看某个 ASR 模型结果；
3. 选择 LLM 模型；
4. 选择或应用提示词模版；
5. 点击“开始提取”；
6. 查看结构化结果和字段对比。

### 6.2 LLM 调用前置条件

必须满足：

- 当前检查记录存在；
- 当前 ASR 结果存在；
- 当前 ASR 结果状态为 `success`；
- 当前 ASR 全文不为空；
- 已选择 LLM 模型；
- 提示词内容包含 `{transcript}`。

### 6.3 LLM 调用逻辑

接口：

```text
POST /api/patients/{exam_record_id}/llm/run
```

请求体：

```json
{
  "llm_model_id": 4,
  "asr_result_id": 12,
  "prompt_content": "..."
}
```

后端流程：

1. 根据 `asr_result_id` 读取 ASR 全文；
2. 根据 `llm_model_id` 创建 LLM 客户端；
3. 将 `{transcript}` 替换为 ASR 全文；
4. 调用 LLM；
5. 解析 JSON 结构化结果；
6. 写入 `patient_llm_results`；
7. 如果当前检查记录有 B 超真实值，则自动计算评估结果；
8. 返回结构化结果。

### 6.4 LLM 结果关联

LLM 结果必须同时关联：

```text
exam_record_id
asr_result_id
llm_model_id
prompt_content
```

这样可以保证：

- 同一检查记录下不同 ASR 结果对应不同 LLM 结果；
- 切换 ASR 模型时，LLM 结果也能对应切换；
- 后续可以比较“同一 LLM + 不同 ASR”的效果。

## 七、提示词模版管理功能说明

### 7.1 功能目标

提示词模版用于管理 LLM 提取规则，避免每次手动复制提示词。

入口：

```text
数据管理 → 检查详情 → LLM 结构化提取 → 模版
```

### 7.2 弹窗布局

模版管理采用左右布局：

```text
提示词模版管理
├── 左侧：模版列表
└── 右侧：模版详情
```

左侧：

- 展示所有提示词模版；
- 支持点击切换；
- 当前选中项高亮；
- 支持新增模版。

右侧：

- 模版名称；
- 是否默认；
- 提示词内容；
- 保存；
- 删除；
- 应用到当前检查。

### 7.3 模版保存规则

保存前校验：

1. 名称不能为空；
2. 内容不能为空；
3. 内容必须包含 `{transcript}`；
4. 名称不能与其他模版重复。

### 7.4 应用到当前检查

“应用到当前检查”只更新当前页面的 `llmPrompt`。

不会立即调用 LLM。

用户仍需点击“开始提取”，才会产生新的 LLM 结果。

## 八、模型配置功能说明

页面：`/model`

功能：

- 新增 ASR / LLM 模型；
- 编辑模型名称、provider、endpoint、api_key、model_name；
- 配置扩展参数；
- 设置启用状态；

模型会被用于：

- 检查详情 ASR 转写；
- 检查详情 LLM 提取；
- 批量实验组合配置。

## 九、批量实验功能说明

### 9.1 功能目标

批量实验用于验证不同模型组合在多条检查记录上的表现。

实验变量包括：

- ASR 模型；
- LLM 模型；
- 提示词模版；
- 热词；
- 检查记录范围。

### 9.2 新建实验

入口：

```text
批量实验 → 新建实验
```

用户填写：

- 实验名称；
- 描述；
- 备注；
- 日期批次；
- 参与实验的检查记录。

当前前端仍以 `selected_patient_ids` 保存病历号列表，后端启动时结合日期解析为检查记录 ID。

后续建议升级为：

```text
selected_exam_record_ids
```

### 9.3 实验组合

进入实验详情后，用户添加实验组合。

组合包含：

- ASR 模型；
- LLM 模型；
- 提示词模版；
- 提示词内容；
- 热词列表；
- 是否启用。

一条实验组合会与每条选中的检查记录生成任务。

### 9.4 启动实验

用户点击“开始实验”后：

1. 后端按日期和病历号解析检查记录；
2. 根据启用组合生成任务；
3. 实验状态变为 running；
4. 后台 Worker 领取任务；
5. 每个任务执行 ASR → LLM → 评估。

### 9.5 实验任务执行逻辑

单个任务维度：

```text
检查记录 × 实验组合
```

执行流程：

1. 读取检查记录音频；
2. 使用组合中的 ASR 模型转写；
3. 保存检查记录级 ASR 结果；
4. 同步写入实验任务 ASR 快照；
5. 如果配置了 LLM，使用组合中的 LLM 和提示词提取；
6. 保存检查记录级 LLM 结果；
7. 同步写入实验任务 LLM 快照；
8. 与 B 超真实值评估；
9. 更新任务状态。

### 9.6 实验快照

`experiment_tasks` 同时保存引用和快照：

| 字段 | 说明 |
|---|---|
| `asr_result_id` | 引用检查记录级 ASR 结果 |
| `llm_result_id` | 引用检查记录级 LLM 结果 |
| `asr_results` | 实验当时 ASR 分段快照 |
| `full_transcript` | 实验当时 ASR 全文快照 |
| `structured_result` | 实验当时 LLM 结构化快照 |
| `evaluation` | 实验当时评估快照 |

保留快照是为了保证历史实验可复现。

### 9.7 实验详情

实验详情展示：

- 实验名称；
- 状态；
- 总任务数；
- 成功/失败/运行中数量；
- 实验组合；
- 检查记录对比表。

对比表包含：

- 病历号；
- 日期；
- 任务状态；
- ASR 转写全文；
- B 超真实值；
- LLM 提取值；
- 字段差异；
- 耗时；
- 错误提示。

### 9.8 实验控制

支持：

- 开始实验；
- 暂停；
- 继续；
- 重试失败任务；
- 删除实验批次。

## 十一、关键数据模型关系

```text
date_folders
  └── patient_records                # 检查记录
        ├── audio_segs               # 录音分段
        ├── b_ultra_results          # B 超真实值
        ├── patient_asr_results      # ASR 持久化结果
        │     └── patient_llm_results # LLM 结果关联 asr_result_id
        └── experiment_tasks         # 批量实验任务
```

### 11.1 ASR 结果表

`patient_asr_results`：

| 字段 | 说明 |
|---|---|
| `patient_id` | 实际为检查记录 ID |
| `record_id` | 病历号快照 |
| `date` | 日期快照 |
| `asr_model_id` | ASR 模型 |
| `segments` | 分段转写 |
| `full_transcript` | 全文 |
| `status` | running/success/failed |
| `is_current` | 是否当前采用 |

### 11.2 LLM 结果表

`patient_llm_results`：

| 字段 | 说明 |
|---|---|
| `patient_id` | 实际为检查记录 ID |
| `asr_result_id` | 关联 ASR 结果 |
| `llm_model_id` | LLM 模型 |
| `prompt_content` | 提示词快照 |
| `structured_result` | 结构化结果 |
| `raw_output` | LLM 原始输出 |
| `evaluation` | 自动评估 |
| `status` | running/success/failed |

## 十二、当前限制与后续优化

### 12.1 当前限制

1. 部分字段仍命名为 `patient_id`，实际语义是检查记录 ID；
2. 批量实验创建阶段仍使用 `selected_patient_ids`，实际存病历号；
3. 前端 TypeScript 类型与 Axios 拦截器返回类型仍有不一致；
4. 提示词模版暂无版本历史；
5. ASR/LLM 历史结果暂未提供完整版本对比视图。

### 12.2 后续优化

1. 新增 `exam_record_id` 字段并逐步替换前端变量；
2. 批量实验选择范围升级为 `selected_exam_record_ids`；
3. 提示词增加版本管理和 diff；
4. ASR/LLM 历史增加对比视图；
5. 增加关键链路自动化测试；
6. 数据库迁移工具化。
