## 迭代概览

当前系统已经从“单条测试工具”演进为“检查记录维度的 ASR/LLM 评估平台”。

关键变化：

- 最小业务维度从“病历号/患者”纠正为“检查记录”；
- 新增检查记录级 ASR/LLM 持久化结果；
- 批量实验任务增加 ASR/LLM 结果引用与快照；
- 数据管理检查详情支持多 ASR 模型切换；
- LLM 提取基于当前选中的 ASR 结果；
- 提示词模版管理升级为左右布局弹窗。

## Round 7：ASR/LLM 持久化运行问题修复

问题：

- 单个 ASR / 批量 ASR 一度均失败；
- ASR SSE 成功调用后结果卡在 running；
- LLM 提取因提示词 `.format()` 与 JSON 花括号冲突失败；
- LLM 记录出现 running 未结束。

修复：

- 单个 ASR 改用 `create_asr`，移除不存在的 `_transcribe_one`；
- 批量实验预加载 `date_folder`，修复 Async SQLAlchemy lazy load；
- ASR SSE 使用独立 `AsyncSessionLocal`；
- 移除 `prompt_template.format(transcript=...)`；
- 前端增加默认 LLM 模型和校验；
- 清理旧 running 记录。
