## 三、核心业务模型

### 3.1 检查记录维度

当前系统中 `patient_records` 实际表示“检查记录”，不是严格意义上的患者主表。

```text
DateFolder
  └── PatientRecord  # 实际为 exam_record
        ├── AudioSeg
        ├── BUltraResult
        ├── PatientAsrResult
        └── PatientLlmResult
```

关键约束：

- `record_id` 是病历号，可跨日期重复；
- `patient_records.id` 是检查记录 ID；
- 所有正式 ASR/LLM 结果都应以 `patient_records.id` 关联；
- 后端部分接口路径仍为 `/api/patients/{patient_id}`，其中 `patient_id` 实际为 `exam_record_id`。

### 3.2 主要表

| 表 | 说明 |
|---|---|
| `date_folders` | 日期目录 |
| `patient_records` | 检查记录，当前命名保留 |
| `audio_segs` | 检查记录下的音频分段 |
| `b_ultra_results` | B 超真实结果，与检查记录一对一 |
| `model_configs` | ASR/LLM 模型配置 |
| `prompt_templates` | LLM 提示词模版 |
| `patient_asr_results` | 检查记录级 ASR 持久化结果 |
| `patient_llm_results` | 检查记录级 LLM 持久化结果 |
| `experiment_batches` | 实验批次 |
| `experiment_combinations` | 实验组合 |
| `experiment_tasks` | 实验任务与结果快照 |
| `test_runs` | 旧单条测试历史，保留兼容 |

## 四、ASR/LLM 抽象层

### 4.1 ASR

统一接口：

```python
async def transcribe(audio_path: str, hotwords: list[str] | None = None) -> str
```

当前支持方向：

- 本地 FunASR；
- MiMo ASR；
- 豆包/火山大模型 ASR；
- 预留讯飞、腾讯等 provider。

实现注意：

- 分段音频逐段识别；
- 最终拼接为 `full_transcript`；
- 热词优先级：接口传入 > 模型配置 > 默认热词；
- 豆包 ASR 采用 full 快照模式，避免增量文本重复拼接。

### 4.2 LLM

统一接口：

```python
async def extract(transcript: str, prompt_template: str) -> LLMResponse
```

当前支持：

- DeepSeek；
- MiMo/OpenAI 兼容接口；
- 预留本地 LLM。

提示词处理要求：

- 使用 `{transcript}` 占位符；
- 不在 `TestExecutor` 中使用 Python `.format()` 处理提示词，避免 JSON 示例花括号导致格式化错误；
- OpenAI 兼容实现中使用安全替换。

## 五、检查记录级 ASR/LLM 持久化

### 5.1 ASR 结果

接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/patients/{exam_record_id}/asr/stream` | SSE 转写并保存 |
| GET | `/api/patients/{exam_record_id}/asr-results` | 查询当前检查记录所有 ASR 结果 |
| GET | `/api/patients/{exam_record_id}/asr-current` | 查询当前采用 ASR |
| PUT | `/api/patients/{exam_record_id}/asr-results/{result_id}/current` | 切换当前 ASR |

保存策略：

- 每次调用 ASR 新增一条 `patient_asr_results`；
- 成功后写入 `segments/full_transcript/status/is_current`；
- 同一检查记录其他 ASR 结果置为非当前；
- SSE 内部使用独立 `AsyncSessionLocal`，避免流式响应期间 DB session 失效。

### 5.2 LLM 结果

接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/patients/{exam_record_id}/llm/run` | 基于指定 ASR 结果执行 LLM |
| GET | `/api/patients/{exam_record_id}/llm-results` | 查询 LLM 历史 |
| GET | `/api/patients/{exam_record_id}/llm-current` | 查询当前采用 LLM |
| PUT | `/api/patients/{exam_record_id}/llm-results/{result_id}/current` | 切换当前 LLM |

请求核心字段：

```json
{
  "llm_model_id": 4,
  "asr_result_id": 12,
  "prompt_content": "..."
}
```

保存策略：

- LLM 结果关联检查记录；
- 同时关联具体 `asr_result_id`；
- 保存原始输出、结构化 JSON、评估结果、准确率。

## 六、提示词模版管理

接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/prompt-templates` | 列表 |
| GET | `/api/prompt-templates/{id}` | 详情 |
| POST | `/api/prompt-templates` | 新建 |
| PUT | `/api/prompt-templates/{id}` | 更新 |
| DELETE | `/api/prompt-templates/{id}` | 删除 |

约束：

- 名称不能为空；
- 内容不能为空；
- 内容必须包含 `{transcript}`；
- 默认模版用于检查详情打开时优先选择。

## 七、批量实验实现

批量实验包含：

```text
ExperimentBatch
  └── ExperimentCombination
        └── ExperimentTask
```

执行链路：

1. 创建实验批次；
2. 选择日期和检查记录范围；
3. 添加实验组合：ASR 模型 + LLM 模型 + 提示词 + 热词；
4. `ExperimentWorker` 领取 pending 任务；
5. `ExperimentRunner` 执行 ASR；
6. 保存检查记录级 ASR 结果，并写入 task 快照；
7. 执行 LLM；
8. 保存检查记录级 LLM 结果，并写入 task 快照；
9. 根据 B 超真实值自动评估。

快照字段保留原因：

- `experiment_tasks.asr_result_id / llm_result_id` 指向检查记录级结果；
- `experiment_tasks.full_transcript / structured_result / evaluation` 保存实验当时快照；
- 历史实验不会因后续切换当前 ASR/LLM 而改变。

## 九、当前技术风险

1. 命名历史包袱：`patient_id` 实际为 `exam_record_id`；
2. 部分旧接口仍按 `record_id` 查询，存在跨日期串数据风险；
3. 前端 TypeScript 类型与 Axios 响应拦截器声明不完全一致；
4. SQLite 迁移依赖脚本，旧表新增列需要显式迁移；
5. SSE 异常中断要确保 running 状态能被清理为 failed。

---

## 十、建议改进

1. 新增 `exam_record_id` 响应字段，逐步替换前端变量命名；
2. 批量实验选择范围升级为 `selected_exam_record_ids`；
3. 为 `patient_asr_results(patient_id, asr_model_id, created_at)` 增加索引；
4. 为 `patient_llm_results(patient_id, asr_result_id, created_at)` 增加索引；
5. 增加前端构建检查和核心 API 测试；
6. 后续数据库迁移工具化，减少手写 ALTER TABLE。
