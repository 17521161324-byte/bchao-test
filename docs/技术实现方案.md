## 三、核心设计

### 3.1 ASR/LLM 抽象层

```python
class BaseASR(Protocol):
    async def transcribe(self, audio_path: str, hotwords: list[str] | None = None) -> str: ...

class BaseLLM(Protocol):
    async def extract(self, transcript: str, prompt_template: str) -> LLMResponse: ...

# 工厂函数
def create_asr(provider: str, **kwargs) -> BaseASR: ...
def create_llm(provider: str, **kwargs) -> BaseLLM: ...
```

新增模型只需实现接口 + 注册 provider，无需改业务代码。

### 3.2 SSE 实时进度

测试执行通过 Server-Sent Events 实时推送进度：

```
event: progress → {"stage": "asr", "current": 3, "total": 12}
event: progress → {"stage": "llm", "status": "processing"}
event: complete  → {"test_id": 42, "status": "ok"}
```

### 3.3 卵泡数解析

输入：`"16.4×1  15.8×1  13.4×1"`
输出：`[{"size": 16.4, "count": 1}, {"size": 15.8, "count": 1}, ...]`

支持 `×`、`﹡`、`*` 三种乘号。

### 3.4 评估逻辑

| 字段 | 容差 |
|------|------|
| 卵泡总数 | 0 |
| 内膜厚度 | ±0.5mm |
| 内膜类型 | 完全一致 |
| 卵巢尺寸 | ±2mm |

## 四、数据模型

- **DateFolder** → **PatientRecord** → **AudioSeg**（录音树）
- **BUltraResult**（ground truth，与 PatientRecord 一对一）
- **ModelConfig**（ASR/LLM 配置）
- **TestRun**（测试记录，含 ASR 结果、LLM 结果、评估）

详见 `backend/app/models/__init__.py`。

## 六、部署步骤

### 6.1 服务器准备

- Windows 系统
- 安装 Python 3.9+（勾选 "Add to PATH"）
- 安装 Node.js 18+
- 安装 PM2：`npm install -g pm2`
- 安装 Nginx

### 6.2 获取代码

```bash
git clone git@github.com:17521161324-byte/bchao-test.git C:\bchao-test
cd C:\bchao-test
```

### 6.3 后端启动

```bash
cd backend
pip install -r requirements.txt
copy .env.example .env
# 修改 .env：RECORDINGS_DIR、LOCAL_ASR_URL 等
pm2 start "uvicorn app.main:app --host 0.0.0.0 --port 8000" --name bchao-backend
pm2 save
pm2 startup
```

### 6.4 前端构建

```bash
cd frontend
npm install
npm run build
# 产物在 frontend/dist/
```

### 6.5 Nginx 配置

1. 复制项目根目录 `nginx.conf` 到 Nginx 的 `conf/` 目录
2. 修改路径：
   - `root C:/bchao-test/frontend/dist;`
   - `alias C:/bchao-test/backend/data/recordings/;`
3. 启动 Nginx

### 6.6 访问

浏览器打开 `http://服务器IP`。
